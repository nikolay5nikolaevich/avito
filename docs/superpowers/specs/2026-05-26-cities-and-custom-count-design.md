# Спецификация: выбор городов (топ-50) + произвольное количество

Дата: 2026-05-26
Статус: одобрено пользователем, готово к реализации

## Цель

Две связанные возможности:
1. **Произвольное количество** объявлений на город — свободный ввод числа вместо
   пресетов (диапазон 1–500, предупреждение при >150).
2. **Выбор городов** — расширить справочник до **топ-50 городов РФ** по убыванию
   населения; на форме дать чекбоксы для выбора, какие города парсить.

Это реализация пункта 6 плана (масштаб) в управляемом объёме (50 городов, не 170).

## Решения, принятые при проработке

1. **Список:** топ-50 городов РФ по убыванию населения. Slug'и выверенные (топ-50
   — известные города, риск неверного slug мал). Данные о населении сверяются
   через веб-поиск при реализации, не «из головы».
2. **Количество:** свободное число, clamp в [1, 500]. >150 — предупреждение
   (дольше; на Авито может не хватить объявлений). Знаменатель среднего = выбранное N.
3. **Дефолт выбора:** при открытии страницы отмечены текущие топ-10 городов.
4. **has_metro:** True только у Москвы и СПб (разбивка по метро по ТЗ только для них),
   даже если у других городов есть метро — в объём не входит.

## Поток данных

```
index.html (input count[1-500] + чекбоксы cities[]=slug)
  → /search: count clamp [1,500]; selected = валидные выбранные slug'и
            если selected пуст → redirect назад
    → run_job(job_id, query, count, filters, selected_slugs)
      → parse_all(query, ..., cities=get_cities_by_slugs(selected_slugs))
        → парсинг только выбранных; progress total = len(selected)
```

## Ключ кэша

`cities_key` = отсортированные выбранные slug'и через запятую (например
`"ekaterinburg,moskva,sankt-peterburg"`). Новый аргумент `cities_key: str = ""`
с дефолтом — обратная совместимость. Полный ключ:
`f"{нормализованный_запрос}|{count}|{filters_key}|{cities_key}"`.

## Валидация и краевые случаи

- `count` вне [1,500] → приводим к границе (без ошибки пользователю).
- Ни один город не выбран → redirect на форму (ничего не парсим).
- Неизвестный slug в форме → отбрасываем (валидация по справочнику).
- Падение/блок одного города при парсинге → лог + пропуск (как сейчас).

## Тестирование (без сети)

- `cities.py` — самотесты: ровно 50 городов, уникальные slug'и, сортировка по
  убыванию населения, `get_cities_by_slugs` (подмножество в правильном порядке,
  игнор неизвестных), `has_metro` только у moskva/sankt-peterburg.
- `cache.py` — изоляция по `cities_key`: один запрос с разным набором городов не путается.
- `tests/smoke_test.py` — POST `/search` с произвольным count и набором выбранных
  городов проходит сквозь; подмена `parse_all` принимает аргумент `cities`;
  `/export.csv` с теми же параметрами попадает в тот же ключ кэша.

## Критерии готовности

- [ ] На форме поле count принимает любое число 1–500, по умолчанию 150.
- [ ] 50 городов чекбоксами по убыванию населения; «выбрать все»/«снять все»; топ-10 по умолчанию.
- [ ] Парсятся только выбранные города; прогресс считает их число.
- [ ] Кэш различает разные наборы городов и разный count.
- [ ] CSV учитывает выбор городов и количество.
- [ ] Все самотесты и smoke-тест зелёные.
- [ ] (Ручная, через CDP) выбор нескольких городов реально парсит только их.

## Вне объёма
