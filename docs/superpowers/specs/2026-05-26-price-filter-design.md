# Спецификация: фильтр цены (расширяемая основа фильтров)

Дата: 2026-05-26
Статус: одобрено пользователем, готово к плану реализации

## Цель

Дать пользователю возможность задавать на сайте фильтры поиска, которые
транслируются в URL Авито и применяются ДО парсинга. Первый фильтр — **цена**
(диапазон «от»/«до»). Архитектура закладывается расширяемой: новые фильтры
(состояние, тип продавца, сортировка и т.д.) добавляются в одном месте без
правки всей цепочки.

В этой сессии реализуется **только цена** + расширяемая основа.

## Решения, принятые при проработке

1. **Объём:** только фильтр цены сейчас; архитектура — под лёгкое добавление будущих.
2. **Ввод цены:** два необязательных числовых поля «от» (`price_min`) и «до»
   (`price_max`). Любое можно оставить пустым (только нижняя, только верхняя, обе
   или ни одной = без фильтра).
3. **Архитектура:** типизированный дата-класс `SearchFilters` (подход C). Вся логика
   «фильтр → параметр Авито» изолирована в одном методе.
4. **Знаменатель метрики не меняется:** среднее по-прежнему делится на выбранное N
   (см. предыдущую фичу — меню количества). С ценовым фильтром объявлений может
   прийти меньше N; среднее честно отражает «плотность спроса» в ценовой полосе.

## Архитектура

### Новый модуль `filters.py`

```python
@dataclass(frozen=True)
class SearchFilters:
    price_min: int | None = None
    price_max: int | None = None

    @classmethod
    def from_form(cls, price_min, price_max) -> "SearchFilters":
        """Создаёт из сырых значений формы с нормализацией (см. «Валидация»)."""

    def to_avito_params(self) -> dict[str, str]:
        """Параметры для URL Авито. Только заданные: {"pmin": "...", "pmax": "..."}."""

    def cache_key_part(self) -> str:
        """Стабильная строка для ключа кэша. Пример: "pmin=1000;pmax=5000".
        Пустые фильтры → "" (обратная совместимость ключа)."""

    def describe(self) -> str:
        """Человекочитаемое описание для шаблонов, напр. "от 1000 ₽ до 5000 ₽".
        Пустые фильтры → "" (в шаблоне ничего не показываем)."""
```

`SearchFilters` неизменяемый (`frozen=True`), как `City`. Поведение единиц
изолировано: `to_avito_params` знает про параметры Авито, `cache_key_part` —
про кэш, `describe` — про вывод. Можно тестировать независимо.

### Поток данных

```
index.html (поля price_min/price_max в форме POST /search)
  → app.py /search: SearchFilters.from_form(...) → объект filters
    → run_job(job_id, query, count, filters)
      → parse_all(query, ..., filters=filters)
        → parse_city(city, query, ..., filters=filters)
          → build_search_url(city.slug, query, filters)  → URL c pmin/pmax
```

Параллельно `filters` участвует в:
- ключе кэша: `save_result/get_result(query, count, filters_key=filters.cache_key_part())`;
- расчёте метрик — не влияет (знаменатель = N);
- контексте шаблонов progress.html / results.html (через `filters.describe()`).

## Параметры Авито (РИСК — подтвердить на живом сайте)

Предполагаем, что Авито принимает цену через query-параметры **`pmin`** и **`pmax`**
(рубли, целые). Это нужно подтвердить при первой живой проверке через CDP
(VPN-цикл). Если параметры иные — правка локализована в `SearchFilters.to_avito_params()`.
URL приобретает вид: `https://www.avito.ru/{slug}?q={запрос}&pmin=1000&pmax=5000`.

## Валидация (в `SearchFilters.from_form`, без ошибок пользователю)

- Пустая строка / нечисловое значение → `None`.
- Отрицательное число → `None` (отбрасываем).
- Если заданы оба и `price_min > price_max` → меняем местами.
- Результат всегда корректный объект; пользователю ошибок не показываем.

## Кэш — обратная совместимость

Новый аргумент `filters_key: str = ""` со значением по умолчанию. Существующие
вызовы без фильтра дают ключ `"{query}|{count}|"` — изоляция от запросов с фильтром
сохраняется, поведение без фильтра предсказуемо.

## Тестирование (без сети)

- `filters.py` — самотесты: нормализация (пусто/нечисло/отрицательное → None),
  swap при min>max, `to_avito_params` (только заданные ключи), `cache_key_part`
  (стабильность и пустой случай), `describe`.
- `cities.py` — тест `build_search_url` с фильтром и без (порядок параметров,
  корректный URL).
- `cache.py` — тест изоляции по `filters_key`: один запрос+count с разными
  ценовыми диапазонами не путается.
- `tests/smoke_test.py` — POST `/search` с `price_min`/`price_max` проходит сквозь
  весь поток; `/export.csv` с теми же параметрами попадает в тот же ключ кэша;
  подмена `parse_all` принимает аргумент `filters`.

## Критерии готовности

- [ ] На главной есть поля «Цена от/до», оба необязательны, уходят в POST /search.
- [ ] `build_search_url` добавляет `pmin`/`pmax` в URL только когда заданы.
- [ ] Кэш различает один запрос с разными ценовыми диапазонами.
- [ ] CSV-экспорт учитывает выбранный диапазон цены.
- [ ] На прогрессе/результатах виден выбранный диапазон цены.
- [ ] Все самотесты и smoke-тест зелёные.
- [ ] (Ручная, на живом Авито через CDP) выбранный диапазон реально применяется на
      Авито — в выдаче только объявления в заданной цене; параметры `pmin`/`pmax`
      подтверждены.

## Вне объёма

Другие фильтры (состояние, тип продавца, сортировка и пр.) — только закладываем
основу, не реализуем. Их добавление — отдельная сессия по тому же шаблону.
