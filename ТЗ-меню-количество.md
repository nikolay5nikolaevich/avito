# ТЗ для следующей сессии: меню выбора количества объявлений

## Контекст
Сейчас количество объявлений на город задаётся только через переменную окружения
`AVITO_MAX_ITEMS` (по умолчанию 150), читается в `app.py` как `MAX_ITEMS_PER_CITY`
и пробрасывается в `parse_all(..., max_items=...)`. В интерфейсе (`templates/index.html`)
выбора нет — только поисковая строка и кнопка «Найти».

Полный прогон 150×10 занимает 1–2 часа, поэтому пользователю нужна возможность
прямо на сайте выбирать, сколько объявлений парсить на город (меньше = быстрее).

**Хорошая новость:** парсер уже готов — `parse_all` и `parse_city` принимают
`max_items`. Менять парсер не нужно. Работа в основном по вебу, аналитике и кэшу.

## Что нужно сделать

### 1. Интерфейс (`templates/index.html`)
- Добавить рядом с поисковой строкой контрол выбора количества объявлений на город.
- Вариант: выпадающий список (`<select name="count">`) с пресетами: **10, 30, 50, 100, 150**.
  По умолчанию выбрано **150**.
- (Опционально) добавить поле для своего числа, но для MVP достаточно пресетов.
- Рядом — подсказка с примерной оценкой времени (например: «150 — полный прогон,
  ~1–2 часа; 30 — быстрее, ~20–30 мин»). Цифры ориентировочные.
- Убедиться, что значение уходит в форму `POST /search` вместе с `query`.

### 2. Бэкенд (`app.py`)
- В обработчике `POST /search` принять новое поле формы `count: int = Form(150)`.
- Провалидировать: целое в диапазоне **1–150** (если вне — привести к границе или
  вернуть понятную ошибку). 150 — текущий разумный максимум.
- Прокинуть `count` в `run_job` → в `parse_all(query, ..., max_items=count)`
  (сейчас там жёстко `MAX_ITEMS_PER_CITY`). `MAX_ITEMS_PER_CITY` оставить как
  значение по умолчанию, если поле не пришло.
- Прокинуть `count` в расчёт метрик (см. п. 3) и в кэш/экспорт (см. п. 4).

### 3. Аналитика — знаменатель среднего (ВАЖНОЕ РЕШЕНИЕ)
В `analytics.py` сейчас среднее = сумма ÷ `TARGET_COUNT` (жёстко 150).

**Проблема:** если выбрать 10 объявлений, а делить на 150 — среднее будет
искусственно занижено (как сейчас в тесте: 5 объявлений → среднее 2.01).

**Решение для реализации:** параметризовать знаменатель — `compute_city_result(city,
listings, target_count=...)`, и передавать туда выбранное пользователем `count`.
Тогда среднее = сумма ÷ N (N = выбранное количество), что корректно отражает выборку.
- Затрагивает: `analytics.py` (константа `TARGET_COUNT` → аргумент функции,
  значение по умолчанию 150 для обратной совместимости), и разбивку по метро
  (там знаменатель — размер группы, не трогаем).
- Обновить самотесты в `analytics.py` под новый параметр.

> Примечание: исходное ТЗ требовало «знаменатель всегда 150». Это имело смысл при
> фиксированных 150. При регулируемом количестве логичнее делить на выбранное N —
> подтвердить это решение в начале сессии.

### 4. Кэш и экспорт (`cache.py`, `app.py`)
Сейчас кэш и `/export.csv` ключуются только по `query`. С регулируемым количеством
это даст неверный результат (например, отдаст кэш на 10 объявлений по запросу на 150).
- Включить `count` в ключ кэша: `save_result`/`get_result` принимают `(query, count)`
  или ключ формируется как `f"{query}|{count}"`. Нормализацию запроса сохранить.
- Эндпоинты `GET /results?query=...` и `GET /export.csv?query=...` должны также
  принимать и учитывать `count` (добавить query-параметр `count`).
- Ссылки «Скачать CSV» в `templates/results.html` — пробросить `count`.

### 5. Отображение
- На странице прогресса/результатов показать, сколько объявлений на город выбрано
  (например, в заголовке: «Запрос: айфон 16 · по 30 объявлений/город»).

## Файлы, которые затрагиваем
- `templates/index.html` — контрол выбора (основное).
- `templates/results.html` — проброс `count` в ссылку CSV, показ выбранного N.
- `app.py` — приём `count` в `/search`, проброс в `run_job`/`parse_all`/аналитику/кэш,
  параметр `count` в `/results` и `/export.csv`.
- `analytics.py` — параметризовать знаменатель (`target_count`).
- `cache.py` — ключ кэша с учётом `count`.
- `parser.py` — **менять не нужно** (`max_items` уже есть).

## Критерии готовности
- [ ] На главной странице есть выбор количества (пресеты, по умолчанию 150).
- [ ] Выбранное число реально ограничивает парсинг (видно по времени и числу карточек).
- [ ] Среднее считается с корректным знаменателем (по выбранному N).
- [ ] Кэш различает запросы с разным количеством (нет «протечки» результата).
- [ ] CSV-экспорт учитывает выбранное количество.
- [ ] На странице результатов видно, какое количество было выбрано.

## Как проверить
1. Запустить debug-Chrome (`start-chrome.bat`) + прогреть avito.ru.
2. `python app.py`, открыть localhost:8000.
3. Выбрать количество = 10, сделать запрос → прогон быстрый, в таблице средние
   адекватные, не заниженные.
4. Повторить с 30 → результат отличается, кэш не путает 10 и 30.
5. Скачать CSV для каждого — числа совпадают с таблицей.
