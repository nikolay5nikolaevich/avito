# Техническое задание: сервис анализа спроса на Авито

## 1. Цель проекта

Веб-сервис, который по поисковому запросу пользователя анализирует объявления на Авито в нескольких крупных городах России и выдаёт таблицу с метриками активности (просмотры за день) по городам и адресам/районам.

**Цель** — понять, в каких локациях наиболее активно интересуются конкретным товаром.

## 2. Этапы разработки

- **Этап 1 — MVP** (то, что делаем сейчас): 10 городов, локальный запуск, базовый функционал.
- **Этап 2 — масштабирование** (после проверки MVP): расширение до 170 городов с населением 100k+, прокси, фоновая обработка.

Всё ТЗ ниже относится к **Этапу 1 (MVP)**.

## 3. Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.11+ |
| Парсинг | Playwright (управление браузером Chromium) + BeautifulSoup (разбор HTML) |
| Веб-сервер | FastAPI + Uvicorn |
| Шаблоны | Jinja2 |
| Кэш | SQLite |
| Запуск | локально (`python app.py` → открыть `http://localhost:8000`) |

Выбор Python обусловлен зрелой экосистемой для скрапинга. Пользователь только запускает команды — писать код самому не придётся.

## 4. Список городов для MVP

Десять крупнейших городов России:

1. Москва
2. Санкт-Петербург
3. Новосибирск
4. Екатеринбург
5. Казань
6. Нижний Новгород
7. Челябинск
8. Самара
9. Уфа
10. Ростов-на-Дону

Москва и Санкт-Петербург получают **расширенную обработку** (разбивка по станциям метро).

## 5. Функциональные требования

### 5.1. Веб-интерфейс

- Одна страница с поисковой строкой
- Кнопка «Найти»
- Индикатор прогресса во время парсинга (показ текущего города: «Парсим Казань… 5/10»)
- После завершения — таблица с результатами
- Возможность скачать результат в CSV

### 5.2. Логика парсинга (по одному пользовательскому запросу)

Для каждого из 10 городов:

1. Открыть страницу поиска Авито с фильтром по этому городу и пользовательским ключевым запросом
2. Собрать ссылки на первые 150 объявлений в выдаче
3. Для каждого объявления открыть его страницу и вытащить:
   - URL
   - Заголовок
   - Цена
   - Адрес / станция метро (как указано на Авито)
   - Просмотров всего
   - Просмотров за день (сегодня)
   - Дата публикации (для расчёта возраста объявления)

### 5.3. Фильтрация объявлений

**Объявление исключается** из анализа, если:

- Опубликовано менее 24 часов назад **И** просмотров от 1 до 30

**Объявление включается**, если:

- Опубликовано более 24 часов назад (любое количество просмотров)
- Или опубликовано менее 24 часов назад, но просмотров 40 и больше

> Граничные значения 31–39 для свежих объявлений: по умолчанию считаем «не включаем». Если хочешь иначе — скажи.

### 5.4. Расчёт метрик

Для каждого города:

- **Среднее** = сумма «просмотров за день» по всем 150 объявлениям ÷ 150
  (если после фильтрации осталось меньше 150 — делим всё равно на 150, как указано в формуле)
- **Топ-3 объявления** по «просмотрам за день» — обязательно с разных адресов/станций метро
  (если в топе три объявления с одной станции — берём следующее по списку с другой станцией)

Дополнительно для Москвы и Санкт-Петербурга:

- Группировка всех 150 объявлений по станциям метро
- По каждой станции метро: среднее «просмотров за день» + топ-3 объявления этой станции
- Адрес определяем по тому, что указывает сам Авито (метро / район / улица — что есть)

### 5.5. Вывод

Таблица, отсортированная по убыванию среднего значения просмотров за день. Структура:

| Город | Адрес/метро | Среднее просм/день | Топ-1 (просм/день, ссылка) | Топ-2 | Топ-3 |
|-------|-------------|--------------------|-----------------------------|-------|-------|

Для МСК/СПб — раскрытие на станции метро (вложенные строки).

## 6. Производительность

- Один запрос пользователя = ~10 городов × 150 объявлений = ~1500 страниц объявлений + 10 страниц выдачи
- Между запросами — задержки (1–3 секунды), чтобы не словить мгновенный бан
- Ожидаемое время на один запрос: **15–40 минут**
- Это локальный MVP без прокси, поэтому возможны капчи и блокировки IP — будем смотреть по факту
- Результаты кэшируются в SQLite на 24 часа: если тот же запрос вбили повторно — отдаём из кэша мгновенно

## 7. Ограничения и риски (честно)

- **Антибот-защита Авито.** Возможны капчи и временные блокировки IP. Решения: реальный браузер через Playwright (а не голые HTTP-запросы), задержки, человекоподобные паузы. Если бан — ждём или используем VPN.
- **Хрупкость селекторов.** Авито периодически меняет вёрстку. Все CSS-селекторы вынесены в один файл `selectors.py` — при поломке правится одна строка.
- **Юридический статус.** Парсинг нарушает пользовательское соглашение Авито. Для личного использования / MVP риски минимальны, но это не для публичного коммерческого запуска.
- **Качество метрики.** «Просмотры объявления» отражают, где находится товар, а не где находится ищущий. Это косвенный, но рабочий показатель активности в локации.

## 8. Структура проекта

```
avito-tracker/
├── app.py                  # FastAPI веб-сервер
├── parser.py               # Логика парсинга Авито
├── cities.py               # Справочник 10 городов
├── selectors.py            # CSS-селекторы (вынесены отдельно)
├── analytics.py            # Расчёт средних и топ-3
├── cache.py                # Работа с SQLite-кэшем
├── templates/
│   ├── index.html          # Страница с поисковой строкой
│   └── results.html        # Страница с таблицей
├── static/
│   └── style.css
├── cache.db                # SQLite (создаётся автоматически)
├── requirements.txt        # Зависимости Python
├── install.sh              # Скрипт первичной установки
└── README.md               # Инструкция как запустить
```

## 9. Что нужно от пользователя

**Один раз при установке:**

1. Установить Python 3.11+ (если ещё нет)
2. Запустить `install.sh` (или последовательность команд из README) — установит зависимости и Playwright
3. Подождать ~5 минут

**Для каждого использования:**

1. Запустить `python app.py`
2. Открыть `http://localhost:8000`
3. Вбить запрос → подождать → получить таблицу

## 10. Критерии готовности MVP

- [ ] Веб-интерфейс открывается локально
- [ ] Поиск по ключевому слову запускает парсинг всех 10 городов
- [ ] Отображается прогресс
- [ ] В выдаче — корректная таблица со средними и топ-3 по каждому городу
- [ ] Для МСК/СПб — раскрытие по станциям метро
- [ ] Кэш работает (повторный запрос — быстрый)
